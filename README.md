## Overview

This repository contains the **source code** for CLuMP and the benchmark algorithms presented in the paper. The associated **data** (pre-generated graphs and results) are archived separately on Zenodo: [![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19048237.svg)](https://doi.org/10.5281/zenodo.19048237).

CLuMP is a novel message-passing-based optimization algorithm designed for spin-glass-like problems defined on sparse graphs (random regular graphs and 2D/3D lattices). It identifies frustrated clusters via belief propagation and proposes collective moves that reduce energy, outperforming standard heuristics such as Simulated Annealing and Extremal Optimization in rugged energy landscapes.

## Repository Structure

```
/
├── README.md
├── LICENSE
└── code/
    ├── Makefile                  # Build all main binaries with a single command
    ├── GraphGenerator.c          # Generate random regular graphs (RRG) and 2D/3D lattices
    ├── GetCluster.c              # Generate clusters at controlled frustration (standalone, not in Makefile)
    ├── CLuMP.c                   # CLuMP optimization algorithm
    ├── RCLuMP.c                  # Population-based variant of CLuMP (R-CLuMP)
    ├── SimulatedAnnealing.c      # Simulated Annealing baseline
    ├── PopulationAnnealing.c     # Population Annealing baseline
    ├── ExtremalOpt.c             # Extremal Optimization (EO) baseline
    ├── ResultsAnalysis.py        # Post-processing and figure generation
    ├── PlotCluster.py            # Visualise clusters generated by GetCluster.c
    └── [header files]            # Shared C headers
```

> **Data** (graphs, raw results, pre-computed outputs for paper figures) are hosted on Zenodo — see the link above.

## Compilation

From the `code/` directory, build all main binaries at once:

```bash
cd code/
make
```

To build a single binary:

```bash
make CLuMP
make GraphGenerator
# etc.
```

To remove all compiled binaries:

```bash
make clean
```

> The Makefile detects your OS automatically and applies the correct flags (`-std=c99` on macOS, `-O2 -Wall` on both platforms). No manual editing required.

## Usage

### Step 1 — Generate a graph

```bash
./GraphGenerator [TYPE] [J] [N] [C]
```

| Parameter | Options | Description |
|-----------|---------|-------------|
| `TYPE` | `2`, `3`, `RRG` | Lattice dimension or Random Regular Graph |
| `J` | `Gaussian`, `PM1` | Coupling distribution |
| `N` | integer | Number of nodes. **Note:** for d=2,3 lattices, `N` is the number of nodes per side |
| `C` | integer | Connectivity (degree). **Note:** for d=2,3 lattices, `C` must be 4 or 6 respectively |

**Example:**
```bash
./GraphGenerator RRG Gaussian 1000 3
```

---

### Step 2 — Run an optimization algorithm

#### CLuMP
```bash
./CLuMP [MAXFRUS] [beta] [log2_iters] [RUN] [graph.conf]
```
| Parameter | Description |
|-----------|-------------|
| `MAXFRUS` | Maximum frustration threshold for cluster selection. Use `AUTO` to automatically adjust `MAXFRUS` to target a 50% BP convergence rate |
| `beta` | Inverse effective temperature (move acceptance probability) |
| `log2_iters` | Total number of iterations as a power of 2 |
| `RUN` | Number of independent runs |
| `graph.conf` | Input graph file |

#### R-CLuMP
```bash
./RCLuMP [MAXFRUS] [R] [log2_iters] [RUN] [graph.conf]
```

| Parameter | Description |
|-----------|-------------|
| `MAXFRUS` | Maximum frustration threshold. Use `AUTO` to target a 50% BP convergence rate |
| `R` | Population size |
| `log2_iters` | Total number of iterations as a power of 2 |
| `RUN` | Number of independent runs |
| `graph.conf` | Input graph file |

#### Simulated Annealing
```bash
./SimulatedAnnealing [betaf] [log2_iters] [RUN] [graph.conf]
```

| Parameter | Description |
|-----------|-------------|
| `betaf` | Inverse final temperature of the annealing schedule |
| `log2_iters` | Total number of MC sweeps as a power of 2 |
| `RUN` | Number of independent runs |
| `graph.conf` | Input graph file |

#### Population Annealing
```bash
./PopulationAnnealing [betaf] [R] [log2_iters] [RUN] [graph.conf]
```

| Parameter | Description |
|-----------|-------------|
| `betaf` | Inverse final temperature of the annealing schedule |
| `R` | Population size |
| `log2_iters` | Total number of MC sweeps as a power of 2 |
| `RUN` | Number of independent runs |
| `graph.conf` | Input graph file |

#### Extremal Optimization
```bash
./ExtremalOpt [log2_iters] [RUN] [graph.conf]
```

| Parameter | Description |
|-----------|-------------|
| `log2_iters` | Total number of iterations as a power of 2 |
| `RUN` | Number of independent runs |
| `graph.conf` | Input graph file |

#### τ-Extremal Optimization

The τ-Extremal Optimization (τEO) results were not produced with this codebase. They were kindly provided by Stefan Boettcher. For a full description of the method, please refer to:

> S. Boettcher, *Extremal optimization: heuristics via coevolutionary avalanches*, Computing in Science & Engineering **2**(6), 75–82 (2002). DOI: [10.1109/5992.881710](https://doi.org/10.1109/5992.881710)

---

### Step 3 — Analyse results

#### Results Analysis

```bash
python3 ResultsAnalysis.py [OUTPUT] [tolerance] [data_file(s)]
```

| Parameter | Options / Notes |
|-----------|-----------------|
| `OUTPUT` | `INFO` (stats only) · `OVERVIEW` (stats + energy evolution) · `FULL` (all + cluster move analysis) |
| `tolerance` | `[0., 1.]` — energy tolerance for convergence. Default `0` (exact match) |
| `data_file(s)` | One or more result files — **all must be in the same folder** |

> **Note for CLuMP/R-CLuMP:** both the result file (`.bp` / `.bpr`) and its energy evolution counterpart (`EVO.bp` / `EVO.bpr`) must be present in the same folder.

---

#### (Optional) Get Clusters

```bash
./GetCluster [MAXFRUS] [seed] [graph.conf]
```

| Parameter | Description |
|-----------|-------------|
| `MAXFRUS` | Maximum frustration threshold. Use `AUTO` to target a 50% BP convergence rate |
| `seed` | *(Optional)* Fixed seed for the random number generator |
| `graph.conf` | Input graph file |

#### (Optional) Plot Clusters

```bash
python3 PlotCluster.py [cluster.conf]
```

Takes the output file of `GetCluster` as input. Visualises internal/external nodes and frustrated loops within the cluster.

## Reproducing Paper Figures

The exact algorithm parameters used to produce each figure are recorded in:
```
data/results/[graph]/[id]/[graph_id]_ALGOspecs.dat
```
*(Download the data archive from Zenodo first.)*

**Quick-start workflow:**
```bash
# 0. Compile
cd code/ && make && cd ..

# 1. Generate graph
./GraphGenerator RRG Gaussian 1000 3

# 2. Run CLuMP
./CLuMP AUTO 0 12 100 graph.conf

# 3. Analyse
python3 ResultsAnalysis.py FULL 0 \
    data/results/[graph]/[id]/*.bp    \
    data/results/[graph]/[id]/*.bpr   \
    data/results/[graph]/[id]/*.sa    \
    data/results/[graph]/[id]/*.eo    \
    data/results/[graph]/[id]/*.taueo
```

| Figure | Data folder | Graph type | Notes |
|--------|-------------|------------|-------|
| Fig. 1 (main) | `data/tests/` | `LRG_2D_N256` | Cluster size vs. frustration |
| Fig. 2 (main) | `data/tests/` | All graphs with Gaussian J, except `LRG_2D_N256` | Cluster size scaling with max-frustration |
| Fig. 3 (main) | `data/results/` | `LRG_2D`, `LRG_3D`, `RRG_C10` | Algorithm comparison — energy vs. time and other stats |
| Fig. 4 (main) | `data/results/` | `LRG_2D`, `LRG_3D`, `RRG_C10` | Collective moves analysis — energy vs. flipped spins |
| Supp. Fig. S1 | `data/tests/` | `LRG_2D_N1024` | Extended SA and PA comparison |
| Supp. Fig. S2 | `data/results/` | `RRG_C4` | Extended algorithm comparison |
| Supp. Fig. S3 | `data/results/` | `RRG_C4`, `RRG_C10` (J = ±1) | Extended comparison with integer couplings |
